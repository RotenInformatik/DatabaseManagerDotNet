Remarks

Scripts holds named scripts as strings. The keys are the batch name, the value is the script.

Callbacks holds named callbacks as delegates. The keys are the batch name, the value is the callback.

TransactionRequirements holds information about transaction requirements for scripts or callbacks. The keys are the batch name, the value is one of the DbBatchTransactionRequirement values. If no value is set for a given batch name, DontCare is used.

Currently, the following options are supported which are extracted from the scripts (see OptionsFormat for more details):

  • TransactionRequirement [optional] One of the DbBatchTransactionRequirement values (as string), e.g. /* DBMANAGER:TransactionRequirement=Disallowed */
 Note
If the TransactionRequirement option is not specified, the value from TransactionRequirements is used. If that is also not available, DontCare is used.
Thread Safety
